# LinearLabel

这些 label 在 `WORKFLOW.md` 里有具体含义：

| Label | 含义 |
| --- | --- |
| `docs` / `documentation` / `readme` | 纯文档改动，只需要最小编译或静态检查。 |
| `ci` / `build` / `test` / `tests` | CI、构建或测试基础设施改动。 |
| `bug` / `defect` / `regression` | 缺陷修复，需要先复现，再验证改动后的流程。 |
| `visual` / `ui` / `frontend` | 视觉或界面改动，需要截图证明。 |
| `gameplay` / `game` | 交互型流程改动，需要运行时媒体证明。 |
| 其他未命中的 label | 走兜底规则，按最宽松的验证要求处理。 |
